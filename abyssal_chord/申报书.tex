\section{项目简介}
深渊协奏是一款以 MoonBit 为主要实现语言的回合制卡牌对战游戏引擎。玩家扮演"调音师"，利用声波武器、护甲系统和永久能力与畸变体战斗。项目包含 \textbf{5,266 行 MoonBit 核心引擎代码}（30个模块），覆盖卡牌战斗、敌人 AI、状态管理、多人对战等完整系统，提供 Next.js Web 可视化前端，支持单机对战与局域网联机。

\section{拟实现的核心功能}
\begin{itemize}[leftmargin=*,nosep]
  \item \textbf{卡牌系统}：55 张卡牌，4 种流派（低频堡垒、过载冲击、高频刺客、回声幻影）
  \item \textbf{战斗引擎}：HP/护甲/AP 管理、5 级污染度系统、声爆 Debuff 叠加引爆
  \item \textbf{敌人 AI}：7 种敌人（含 2 Boss 二阶段），骰子驱动行动选择，4 级难度（简单/普通/困难/噩梦）
  \item \textbf{永久能力}：频率锚定、低频共振、痛觉回响、终末定音、回声标记、全频共振
  \item \textbf{游戏模式}：快速对战（10种预设）、生存模式（无尽波次）、Boss Rush、训练模式、本地联机
  \item \textbf{辅助系统}：卡牌数据库、套牌构建器、成就系统（40项）、战斗回放、AI vs AI 平衡性模拟器
  \item \textbf{工程质量}：108 个自动化测试（全部通过）、GitHub Actions CI/CD（check+build+test）
\end{itemize}

\section{原创性说明}
本项目为\textbf{原创项目}。卡牌体系、数值模型、污染度系统、声爆机制均为自主设计。MoonBit 引擎代码全新编写，TypeScript 前端基于 Next.js 生态组件构建。数值平衡模型：1AP = 5伤害 = 5护甲，超模设计通过副作用（自伤/污染代价/消耗）平衡。

\section{技术路线}
\begin{itemize}[leftmargin=*,nosep]
  \item \textbf{核心引擎}：MoonBit 语言，利用类型系统、模式匹配、枚举等特性实现复杂游戏逻辑
  \item \textbf{AI 工具}：Claude Code 辅助代码生成、Bug 修复、测试编写、卡牌平衡分析
  \item \textbf{前端}：Next.js + React + Framer Motion 动画，WebSocket 多人实时同步
  \item \textbf{测试}：108 个黑盒测试覆盖污染度、卡牌数据、战斗状态、伤害结算、胜负判定
  \item \textbf{CI/CD}：GitHub Actions 自动运行 \texttt{moon check + build + test}
\end{itemize}
